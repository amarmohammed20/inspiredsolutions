Seismi

Consultant

A consultant implementing complex Oracle ERP solutions for many large corporate clients. My role entailed implementing Oracle applications such as Oracle Data Relationship Management and Oracle Financial Data Quality Management. Oracle DRM was used to produce one standardized view for master data while ensuring validations were applied through rules and properties. Oracle DRM was also used as the central hub for sending and receiving cleansed master data between many planning and reporting systems. Oracle DRM allowed me to use governance/workflows to direct change within a business to the relevant business people while ensuring validations were met, audit trails were automatically created and a standardised process was consistently followed. Using JavaScript code, I created and maintained sophisticated properties and validations, enriching the data for downstream and upstream systems. Overall I customised the standard Oracle product and tailoring the DRM package to exceed the client expectations.

Oracle FDMEE was used as an ETL tool that would very neatly integrate into other Oracle tools such as HFM and Hyperion Planning. This ETL tool enabled me to extract data using data load rules with particular filters to ensure only the required data was extracted. Complex mappings (Explict, Like and Multi-Dimensional) were applied enabling the organisation to transform this data before any validations were ran within FDMEE. Finally loading this data into an Oracle Hyperion Planning application ready for the Planning & Reporting team. Through the setup and development of FDMEE event scripts were created using the coding language Jython. These Jython scripts enabled me to push the mapped data into the Oracle Enterprise Data Warehouse (EDW).
